How To Use


1. Getting Started

If you want to raid a server, you need tokens and proxies (optional).

Tokens are the accounts that will be used to raid the server. You can either create them yourself or buy them from someone. Never buy tokens priced above $0.10 per token, as they are overpriced. Always buy email- or phone-verified tokens.

Proxies are used to hide your IP address so you don't get rate-limited. They also increase the performance of the tool. I recommend buying proxies from iproyal.com. After getting all the required items, you need to configure them on the settings page.


2. Using The Modules

q To use a module, you need to fill in the required fields and press the start button. The module will start, and you will see the results in the console. You can stop the module by returning to the dashboard.


3. Raiding Tutorial

To raid a server, you need to join it first. You can use the server joiner module for that. Before starting the spammer or any other module, make sure the server you are joining doesn't have any verification system. If it does, you can use the bypass modules to bypass them.


4. Status Codes

Status codes are the responses from the Discord API. Each status code is different and has a distinct meaning. Here are some of the most common status codes:
200, 201, 204 status codes mean that everything went well.
400 status code is usually interpreted as "bad request," but 90% of the time it just means that your tokens encountered a captcha.
401 status code means that the token you are trying to use is invalid.
403 status code means that you don't have permission to perform an action. When receiving this error code, make sure you have passed the verification on the server properly.
429 status code means that you got rate-limited. Don't panic; rate limits usually disappear after a few hours. Use proxies or a VPN to avoid getting rate-limited.
